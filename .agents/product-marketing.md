# Vavist Product Marketing Context

## Site

- Name: Vavist
- Production URL: `https://vavist.com`
- Type: root-domain editorial hub and launchpad for Three.js Lab
- Lab URL: `https://threejs.vavist.com`
- Audience: Three.js/WebGL builders researching practical scene problems
- Goal: build trust, answer guide/search questions, route users to the right lab tool

## SEO Positioning

Own guide, checklist, troubleshooting, and workflow queries:

- `three js guides`
- `webgl scene health check`
- `three js gltfloader checklist`
- `three js responsive canvas`
- `three js mobile performance checklist`
- `three js color management`
- `three js scene debugging checklist`
- `three js loading progress`
- `three js static site seo`

Avoid duplicating exact utility landing queries owned by `threejs-lab`, such as `online glb viewer` or `camera fov calculator`. Instead, explain when to use the lab tool and link to it.

## Page Quality Rule

Every guide should include:

- One precise H1.
- Article JSON-LD, FAQ JSON-LD, and breadcrumb JSON-LD.
- Original explanatory sections.
- Source links to primary documentation.
- Related guide links.
- A clear next step toward a lab tool or checklist.
